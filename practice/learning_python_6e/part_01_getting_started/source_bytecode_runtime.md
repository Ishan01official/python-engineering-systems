# Source, Bytecode, Runtime

This note explains what happens when you run:

```bash
python practice/learning_python_6e/part_01_getting_started/hello_script.py
```

## 1. Terminal Command

The shell starts the Python interpreter:

```text
python
```

and passes this script path as input:

```text
practice/learning_python_6e/part_01_getting_started/hello_script.py
```

If you use the absolute path, it is:

```text
/home/ishan/python-engineering-systems/practice/learning_python_6e/part_01_getting_started/hello_script.py
```

## 2. Source Code

The `.py` file is source code: human-readable Python text.

For `hello_script.py`, Python reads the file and sees:

```python
def main():
    message = "Python is running from a script."
    print(message)


if __name__ == "__main__":
    main()
```

plus your extra practice lines:

```python
print("hello world")
print(2**100)
```

## 3. Bytecode

Python does not run source text directly. It compiles source code into bytecode instructions first.

Important detail:

- Imported modules usually get cached in `__pycache__/`.
- A directly executed script is compiled for execution, but Python does not always keep a `.pyc` cache for the main script.
- The demo file `execution_model_demo.py` uses `py_compile` so you can see an actual `.pyc` file.

Run:

```bash
python practice/learning_python_6e/part_01_getting_started/execution_model_demo.py
```

Then check:

```bash
find practice/learning_python_6e/part_01_getting_started -name "*.pyc" -print
```

## 4. Runtime

Runtime means Python is now executing code objects.

For this block:

```python
if __name__ == "__main__":
    main()
```

Python checks the special variable `__name__`.

When a file is run directly:

```python
__name__ == "__main__"
```

So Python calls:

```python
main()
```

## 5. Variable Location

In Python, a variable is not a fixed memory box like in C.

A variable is a name inside a namespace:

- global variables live in the module namespace, visible through `globals()`
- local variables live in a function frame while the function is running, visible through `locals()` or `frame.f_locals`

Example:

```python
message = "Python is running from a script."
```

This creates or reuses a string object, then binds the local name `message` to that object.

In CPython, `id(message)` is the memory address of the object for that process run. It can change every time you run the program.

## 6. Mental Model

```text
terminal command
    -> Python interpreter starts
        -> reads .py source file
            -> compiles source to bytecode
                -> creates code objects
                    -> executes module code
                        -> defines functions
                        -> calls main()
                            -> creates local variables
                            -> prints output
```
